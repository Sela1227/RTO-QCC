## 一、萃取的設計 context

- **這 app 做什麼：** 追蹤放射治療病人療程期間的體重變化，跌破門檻自動警示並觸發營養師轉介
  （來源：CLAUDE.md §〇 當前狀態 + §三 業務對映表）
- **給誰用：** 放射腫瘤科醫護人員（醫護端）；病人自我回報體重（病人端）
  （來源：CLAUDE.md §〇 頁面架構）
- **解決什麼痛點：** 病人因治療副作用悄悄掉體重，若沒被及時發現會影響療程完成率；系統要在「掉太多之前」就叫出來
  （來源：CLAUDE.md §三 警示閾值 -3% SDM / -5% 營養師 / 7 天處置窗）
- **情緒基調：** 嚴謹、安靜、可信賴 —— 不是活潑或行銷感。這是臨床工具，錯過警示是病人安全問題
  （來源：CLAUDE.md §一 技術棧決策「病人資料完全留瀏覽器本機」+ §七 下版候選第 1 名的措辭）
- **使用情境：** 醫護每天在診間 / 護理站開著，快速掃出「誰體重掉太多、誰待輸體重、誰需要介入」
  （來源：CLAUDE.md §三 追蹤清單 8 Tab）

## 二、自動決策

### 範本類型
- **選定：** B — 醫療專業型
- **理由：** 放射腫瘤科臨床工具，處理病人體重與營養介入，屬醫療 / 臨床 / 健康類（§17.2 決策 1）

### 壁虎/蜥蜴傾向
- **傾向：** not a natural fit here —— **prompt 中不提壁虎**
- **理由：** 依 §13.2 流程走 —
  - Q1「守護 / 持續存在」精神：**契合**（系統確實是靜默守著病人體重曲線）
  - Q2「跟 SELA 主 logo 並排會混淆嗎」：**會**。app logo 若也是「純色底 + 白色壁虎剪影」，跟 SELA 橘底白壁虎只差顏色，在 README（app logo 在頂部、SELA 微標在 footer）會讓人分不清是主品牌還是子產品
  - 依 §14.7 守則「不契合就根本別提、不要寫『壁虎不適合但你看著辦』」→ prompt 完全不提，主體交給生圖 AI 發揮

### 背景色提案（圓角填色、背景是白底待後製）
- **選定：** 北歐霧藍 `#5B8FB9`
- **理由：**
  - §17.2 決策 3 Step 1：醫療專業 → 北歐霧藍 ✓
  - `#5B8FB9` 是本專案**長期使用者驗證的標準色**（CLAUDE.md 衝突仲裁區塊列為「不做」等級、不可更動），app logo 用它可與系統 UI、PWA `theme_color` 一致
  - Step 2/3 撞色檢核：非愛馬仕橘 → 不與 SELA 主 logo 撞色，免走 Step 3

## 三、完整 Prompt（複製這段貼到生圖 AI）

```
一個 app icon 的 logo。

這個 app 是什麼：
這是一個放射線治療病人的體重監控系統。治療期間病人常因副作用持續掉體重，
系統追蹤每次量測的體重變化率，跌破門檻就自動警示、轉介營養師。
醫護每天在診間開著，快速掃出誰的體重掉太多、需要介入。

設計要求：
- 圓角方形、1:1 比例
- 扁平向量風格、單一白色主體圖案
- 底色填 #5B8FB9、放在純白背景上
- 不要漸層、陰影、3D、光暈
- 小尺寸（favicon）也要清楚

主體畫什麼由你發揮、呼應這個 app 的精神即可。
```

## 四、給 SELA 的備註

- **prompt 已要求 AI 輸出乾淨純白底 PNG**（Gemini 生不出透明背景，白底反而好後製）
- 拿到原圖後 → 回我這邊走 `logo/CLAUDE.md` §10.2 四步（取底色 → 四角 floodfill → 滿版方形 → 圓角交顯示端）把白底轉透明 + 生多解析度套組，再套進 `favicon/` 與兩份 manifest
- prompt **刻意不指定畫什麼主體** —— 只讓 AI 認識這個 app、自己發揮。如果生出來的方向不喜歡 → 跟我說，我可在 prompt 補「avoid X」或「lean toward Y」微調，但仍不寫死主體
- 如果 prompt 本身不滿意 → 跟我說，我重生（不必改 CLAUDE.md，回頭看 CLAUDE.md 就能重跑）
- 想改範本類型 / 顏色 / 壁虎傾向 → 告訴我「改 X 為 Y」，我重產這份檔
