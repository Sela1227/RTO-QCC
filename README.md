# RTO-QCC 彰濱放腫體重監控預防系統

放射線治療病人體重監控與營養介入管理系統。

## 系統特色

- **純網頁應用**：無需安裝，瀏覽器直接使用
- **本機儲存**：資料存於瀏覽器 IndexedDB，保護病人隱私
- **跨平台相容**：Windows / macOS / iOS / Android 皆可使用
- **北歐簡約設計**：清爽介面、大量留白、低視覺負擔

## 快速開始

### 方法一：直接開啟

雙擊 `index.html` 即可使用。

### 方法二：本機伺服器（推薦）

```bash
cd RTO-QCC
python -m http.server 8000
# 開啟 http://localhost:8000
```

### 方法三：部署至 GitHub Pages

1. 建立 GitHub Repository
2. 上傳所有檔案
3. 前往 Settings → Pages → 選擇 main branch
4. 等待幾分鐘後即可使用

## 功能總覽

### 醫護端（index.html）

| 模組 | 功能說明 |
|------|----------|
| 儀表板 | 今日摘要、統計卡片、快速操作 |
| 治療清單 | 追蹤中/暫停中/需處理/需關注/待輸體重 分類顯示 |
| 資料庫 | 病人查詢、多重篩選（區間/狀態/癌別/醫師）、多重排序 |
| 統計報表 | 療程/體重/介入/人口/副作用/滿意度 統計、Excel/PDF 匯出 |
| 系統成效 | 8 項 KPI 指標、5 張分析圖表、PDF 報告匯出 |

**核心功能**：
- 病人管理：新增、編輯、搜尋、軟刪除
- 療程管理：新增、暫停、恢復、結案、終止
- 體重追蹤：記錄、編輯、趨勢圖、線性預測
- 警示機制：SDM 閾值 -3%、營養師閾值 -5%、7 天內處置判斷
- 介入管理：轉介、聯繫、執行、跳過、反應時間追蹤
- 副作用評估：CTCAE v5.0 標準、8 項症狀、疼痛 0-10 量表
- SDM 營養選擇：口服營養、鼻胃管、胃造廔術比較表
- 滿意度調查：5 題問卷、NPS 淨推薦值計算
- 資料備份：匯出/匯入 JSON、多機同步合併
- QR Code：產生給病人掃描、匯入病人回傳資料

### 病人端（patient.html）

| 頁籤 | 功能說明 |
|------|----------|
| 體重 | 每日體重記錄、歷史紀錄、趨勢圖表 |
| 副作用 | 症狀自我評估、疼痛量表 |
| 滿意度 | 服務滿意度問卷、文字回饋 |
| 設定 | 提醒時間、回傳 QR Code |

**使用流程**：
1. 掃描醫護端產生的 QR Code 初始化
2. 每日記錄體重和副作用
3. 療程結束填寫滿意度
4. 產生回傳 QR Code 給醫護端匯入

## 資料儲存

資料存於瀏覽器 IndexedDB（共 7 個資料表）：

| 資料表 | 內容 |
|--------|------|
| patients | 病人基本資料 |
| treatments | 療程資料（含主治醫師、期別、SDM 選擇）|
| weight_records | 體重記錄 |
| side_effects | 副作用評估 |
| interventions | 介入記錄 |
| satisfaction | 滿意度調查 |
| settings | 系統設定 |

**注意事項**：
- 關閉瀏覽器不會遺失資料
- 清除瀏覽器資料會遺失所有內容
- 建議定期使用「設定 → 匯出備份」

## 多機同步

1. A 電腦：設定 → 匯出備份 → 下載 JSON
2. 複製檔案至 B 電腦
3. B 電腦：首頁 → 同步 → 選擇檔案
4. 系統自動比對並合併資料（以較新資料為準）

## 瀏覽器支援

- Firefox 75+
- Safari 14+
- Edge 80+

